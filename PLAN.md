# zig-libs — execution plan

Sequenced build plan. Rule: **complex/high-value first, then extractions of what we already
have or will soon improve.** Dependency order forces a few foundations early (noted).
Catalog + per-module descriptions: `zig-libs-plan.md`. Context/rules: `BRIEF.md`.

Definition of done (every task): module folder created, `pub const meta` filled, public API +
doc-comments, tests, registered in `build.zig`, `zig build test-<name>` **and** `zig build test`
green (Debug + ReleaseFast), `zig fmt --check` clean, `README.md` written. Where an oracle exists
(h2spec / golden bytes / live round-trip), run it.

> Note for agents: **do NOT edit this file** — the repo owner maintains Progress. Just report back.

## Wave P0 — complex & high-value

- **T1 `netaddr`** — RFC 6724 address selection (foundation for http/dns). ✅
- **T2 `http`** — Ph1 client over TLS ✅ · Ph2 server codec ✅ · Ph2.1 hardening ✅ · **Ph3 HTTP/2 (framing+HPACK, h2spec) = later**.
- **T3 `dns` + DoH** ✅
- **T4 `netlink`** — rtnetlink transport (routes/links/neighbors); model libmnl/libnl; netns verify. **pending**
- **T5 REST/API cluster** (direct-internet, max-security) — `router` ✅ · `ratelimit` ✅ · `abuseguard` ✅ · `throttle` ✅ · `security-headers` ✅ · `cors` ✅ · `metrics` ✅ · **`validate` (T5.9)** · `resilience` · (router route-enumeration) → `openapi`.
- **T6 `kv`** — xitdb-style embedded KV + VOPR sim. **pending**
- **T7 `finstats` (advanced)** + **`exprcalc` (capstone)**. **pending**

Later additions for direct-HTTPS: **TLS-terminating server** + **gzip response**.
